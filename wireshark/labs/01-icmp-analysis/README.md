# Lab 01: ICMP Analysis

## Status

Planned

## Objective

Capture and analyze ICMP traffic generated by the `ping` command,
identifying Echo Request and Echo Reply messages, and observing fields
such as IP addresses, TTL, ICMP type, and code. It also aims to
observe possible signs of packet loss, elevated latency, or abnormal
behavior in the communication.

## Concepts covered

- ICMP protocol (Internet Control Message Protocol).
- Echo Request (type 8) and Echo Reply (type 0) messages.
- ICMP error messages (e.g., Destination Unreachable — type 3, Time
  Exceeded — type 11).
- TTL (Time to Live) field of the IP header.
- Relationship between `ping` and the ICMP protocol.
- Capture and display filters specific to ICMP.
